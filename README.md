# ESP32-S3 WiFi Audio Receiver + Android Streamer

Захват системного звука Android (YouTube/Spotify/любое приложение) и
трансляция по WiFi UDP на ESP32-S3 с выводом на MAX98357A.

```
Android (YouTube/Spotify)
   ↓ AudioPlaybackCapture (системный звук)
   ↓ UDP PCM 16bit/44.1kHz stereo
WiFi AP "ESP32-Audio"
   ↓
ESP32-S3 (192.168.4.1:3333)
   ↓ I2S
MAX98357A → 🔊 динамик 4-8Ω
```

## Подключение MAX98357A к ESP32-S3 Super Mini

| MAX98357A | ESP32-S3 | Примечание |
|-----------|----------|------------|
| VIN       | 5V       | берём с USB-пина 5V, или отдельное питание |
| GND       | GND      | |
| BCLK      | GPIO4    | I2S bit clock |
| LRC       | GPIO5    | I2S word select |
| DIN       | GPIO6    | I2S data |
| SD        | —        | можно не подключать |
| GAIN      | —        | не подключать = +9дБ |
| + / –     | динамик 4–8 Ω, 3Вт |

Если 5V с платы не хватает (писк, треск) — кинь 470мкФ конденсатор между VIN и GND у самого MAX98357A, либо запитай отдельно.

## Прошивка ESP32-S3

1. Arduino IDE → плата: **ESP32S3 Dev Module**
2. `Tools → USB CDC On Boot: Enabled` (чтобы виделся Serial монитор)
3. `Tools → Partition Scheme: Default 4MB with spiffs` (размера хватает)
4. Открываешь `esp32_sketch/esp32_audio_receiver.ino`
5. Заливаешь

В Serial Monitor (115200) должно появиться:
```
=== ESP32-S3 WiFi Audio Receiver ===
AP started: SSID="ESP32-Audio" PASS="12345678"
IP: 192.168.4.1   UDP port: 3333
I2S initialized (MAX98357A)
Ready. Подключай телефон к WiFi и запускай приложение.
```

## Сборка Android-приложения

**Требования:** Android Studio (любая свежая), телефон с Android 10+.

1. Открываешь Android Studio → `Open` → папка `android_app`
2. Ждёшь пока Gradle синхронизируется (~2-5 минут первый раз)
3. Подключаешь телефон по USB, включаешь отладку по USB
4. Кнопка **Run** → приложение установится и запустится

Либо собрать APK: `Build → Build Bundle(s)/APK(s) → Build APK(s)` → полученный APK перекинуть на телефон и установить.

## Использование

1. Включаешь ESP32 — появляется WiFi сеть **ESP32-Audio** (пароль `12345678`)
2. На телефоне подключаешься к этой WiFi (интернета не будет — это нормально, система спросит "сохранить подключение без интернета?" — "да")
3. Запускаешь приложение, IP и порт уже вписаны (192.168.4.1 : 3333)
4. Жмёшь **Старт** → разрешаешь захват экрана/звука
5. Включаешь YouTube, Spotify, любой плеер — звук идёт на колонку

## Параметры, которые можно менять

**В скетче:**
- `AP_SSID`, `AP_PASSWORD` — имя и пароль WiFi
- `I2S_BCLK`, `I2S_LRC`, `I2S_DIN` — пины DAC
- `SAMPLE_RATE` — можно поднять до 48000, если телефон шлёт 48к

**В приложении:**
- IP и порт вводятся в UI
- Размер UDP пакета — в `AudioCaptureService.streamLoop()`, константа `pktSize`
- Список захватываемых категорий — `addMatchingUsage(...)` там же

## Типичные проблемы

**Нет звука, в Serial лог `[idle]`** — телефон не шлёт пакеты:
- Проверь что подключен к WiFi `ESP32-Audio`, а не к другой сети
- Проверь IP в приложении — должен быть 192.168.4.1
- Приложение запущено (уведомление "Audio Stream" висит)
- Что-то реально играет на телефоне

**Звук с заиканиями/щелчками:**
- Увеличь `dma_buf_count` в скетче с 8 до 16
- Уменьши `pktSize` в приложении с 1440 до 960
- Поднеси телефон ближе к S3 (тонкая антенна на S3 Super Mini слабовата)

**Большая задержка (~1 сек):**
- Это нормально для UDP + DMA буферов. Для лайв-игр не годится, для музыки ок.
- Уменьшить можно `dma_buf_len` (до 128) и `pktSize` (до 480 = 2.7мс)

**Перегрев / писк MAX98357A:**
- Плохое питание 5V. Поставь конденсатор 470-1000мкФ между VIN и GND у модуля.

**Не захватывается звук приложения:**
- Приложение установило `FLAG_SECURE` или `ALLOW_CAPTURE_BY_NONE` (банковские, Netflix с DRM). Такие не захватить — это системное ограничение Android.

## Защищённость и лимиты

- WPA2 с паролем `12345678` — поменяй на нормальный
- До 4 клиентов в AP одновременно (S3 по умолчанию)
- Без фоновой интернет-раздачи — ESP32 работает только как AP для звука
